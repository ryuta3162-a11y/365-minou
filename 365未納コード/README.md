# 365未納コード（GAS・ローカル作業用）

FIT365 / JOYFIT の未納管理スプレッドシート用 Apps Script です。店頭向け POP はリポジトリ直下の **`365未納POP/`** を参照してください。

このフォルダには **ブランド別に独立した 2 つの Apps Script 用ソース**があります。スプレッドシート側では **プロジェクトを分けて** 紐付けてください。


| フォルダ        | 用途                      | 主なファイル                   |
| ----------- | ----------------------- | ------------------------ |
| **FIT365/** | FIT365 未納・請求まわり         | `Code.gs`, `dialog.html` |
| **JOYFIT/** | JOYFIT（ジョイフィット）未納・請求まわり | `Code.gs`, `dialog.html` |

**本番との比較用**（志木店などエディタから貼り付け）: `FIT365/本番スクリプト貼付/`

ルート直下に `Code.gs` は置かない構成にしています（どちらのプロジェクトか迷わないため）。

## 各店ブックへ Code.gs / dialog.html を貼り直したあと（8店舗など）

1. スプレッドシートを開き直し、メニュー **「FIT365 各店（請求・同期）」→「初回セットアップ」** を1回実行（権限・トリガーまとめて設定）
2. 6月タブがまだ無い／非表示の店は **「📅 当月シートを手動作成」** または非表示シートの表示
3. 未納データは **「未納者データ自動生成 (2CSV)」** は従来どおり手動

※ GAS を貼るだけではトリガーは直りません。貼り直し後も **「初回セットアップ」** を店舗ごとに1回ずつ実行してください。

## 全店ブック（`FIT365_全店実施管理ブック_GAS/Code.gs`）

メニュー **「FIT365 全店」**（ポップアップは「完了しました」「エラーのみ）

| メニュー | 誰が | 用途 |
| --- | --- | --- |
| **初回セットアップ** | 管理者（1回） | 全店ブックの権限・編集時トリガー |
| **更新** | 月初担当 | `26年6月` 等のタブを開いた状態で、各店の実施状況を取り込み＋SMS日程を各店へ配信 |
| **リマインド送信** | リマインド担当 | 本日分を手動送信 |
| **リマインド自動設定** | リマインド担当（1回） | 時間トリガー `fit365ReminderDailyTrigger`（毎朝・**送信者の Gmail**） |

**役割の整理**

- **各店8冊** … 管理者が `Code.gs` 貼付＋初回セットアップ＋トリガー（店舗はシートのみ）
- **全店1冊** … 管理者が初回セットアップ。月次タブは手動複製
- **リマインド** … 全店と別。連絡先シートを使う人が「リマインド設定」（管理者が代行すると管理者メールから送信される）

月次タブ（6月・7月）は **手動で前月シートを複製** してから **更新**。

※ メニューが「全店同期セットアップ」「同期テスト」等のままなら、全店用 `Code.gs` の貼り替え後にシートを開き直してください。
